## 🚨 GO / NO-GO blockers — fix before any paying medical practice goes live

These would block customer #1 from a HIPAA-conscious practice or operator. None are in our P0 roadmap; they are gaps in what we already shipped.

### B1. Railway does not offer a BAA (HIPAA #1)
**HIPAA audit, top finding.** Railway hosts the application + the PostgreSQL database holding all PHI. Without a signed Business Associate Agreement, *every* paying customer is in technical violation of HIPAA. **Action:** evaluate Aptible, Render-with-BAA, or AWS-eligible (RDS + ECS) before onboarding paid customer #1. This is a multi-week migration project.

### B2. Patient PHI fields stored in plaintext
- `Patient.email`, `phone`, `address`, `city`, `state`, `zip`, `gender`, `date_of_birth` — 18 demographic identifiers — **not encrypted at rest**.
- `Encounter.subjective`, `objective`, `assessment`, `plan`, `diagnoses`, `vitals`, `screening_scores`, `structured_data` — **the entire SOAP note** — not encrypted.
- `Prescription.medication_name`, `dosage`, `frequency`, `dea_number`, `notes` — not encrypted.
- `LabOrder.panels`, `diagnosis_codes`, `notes` — not encrypted.
- `Document.name`, `original_name`, `description` — not encrypted.

**Action:** add `encrypted` casts + write a data migration that re-encrypts existing rows. Single biggest HIPAA exposure. Touched in HIPAA + Database audits.

### B3. Cross-tenant PHI leak via Messages
`MessageController.php:71-117` — `recipient_id` is validated only as `exists:users,id`. A practice_admin in tenant A can send a message to any user in tenant B, and that user reads it via their own `recipient_id` filter. **Active PHI exfiltration channel.** Fix: scope the `exists` rule to `tenant_id = $user->tenant_id`. **5-minute fix.**

### B4. Public enrollment can be plan-id-spoofed across tenants
`ExternalController.php:138` — public enrollment endpoint validates `plan_id` exists but doesn't scope to the URL's `tenantCode`. A patient enrolling at Clinic A can be issued a membership tied to Clinic B's plan. Corrupts revenue reporting and creates orphan rows. **5-minute fix:** scope the `MembershipPlan::findOrFail` to the resolved practice.

### B5. SMS webhooks are unauthenticated
`/api/webhooks/sms/inbound` and `/status` accept any caller — no Twilio signature verification. Anyone can inject inbound SMS as if from a patient's phone, polluting message threads or marking prescription-status events. **Fix:** validate `X-Twilio-Signature` against `Twilio\Security\RequestValidator`.

### B6. PHI leak in PatientPortal mock fallback
`frontend/src/components/portals/PatientPortal.tsx:670-675` — when a real patient has no data yet, mock fields fill in showing **"James Wilson, Complete Plan, MBR-284719, Dr. Nageley Michel"** on their membership card. Real patients see another fictional person's PHI. **Fix:** initialize state to `[]` not `null`, drop `||` mock fallbacks, source identity from auth user.

### B7. Kiosk endpoints are unauthenticated PHI dispensers
`KioskController` — after a 4-digit PIN match (no per-PIN lockout, no length validation), all subsequent kiosk endpoints (`/screenings`, `/consents`, `/check-in`) require only `tenant_code` + `patient_id` (UUIDs leak via referrer). Anyone can pull pending screenings + unsigned consents for any enumerated patient. **Fix:** `identify` must mint a 5-min kiosk session token; subsequent endpoints verify it; per-PIN lockout after 5 fails.

### B8. Telehealth `room_url` exposed to any tenant member
`TelehealthController::show/join` only checks `tenant_id`. A staff member in tenant A can pull any other patient's Daily.co join URL — meaning they can join the call. **Fix:** assert caller is the appointment's patient, provider, or `practice_admin`.

### B9. AppointmentBookingWidget is 100% mocked
`AppointmentBookingWidget.tsx:46-100, 148-177` — providers, types, slots are hard-coded constants. `handleBook()` is `setTimeout(1200)` and never calls the API. Patients clicking "Book" see a success screen but **no appointment is created server-side**. This widget is wired into PatientPortal home. **Fix:** replace mocks with `appointmentTypeService`, `providerService`, `availabilityService`, `appointmentService.create`.

### B10. Patient telehealth navigation is broken
`PatientPortal.tsx:1014, 1163` — clicking "Join Video" navigates to `/telehealth/session-${apt.id}` (literal string concatenation), not a real session UUID. **Fix:** call `telehealthService.createSession({ appointmentId })` first, navigate to returned id.

### B11. Audit logs and PHI access logs are mutable
`AuditLog`, `PhiAccessLog`, `SecurityEvent` are standard Eloquent models. `AuditLog::truncate()` from any DB session works; any controller can `$log->delete()`. HIPAA §164.316(b)(2)(i) requires 6-year retention; SOC 2 CC7.3 requires integrity. **Malicious insider can erase tracks.** Fix: override `delete`/`update` to throw; create read-only DB role for app, write-only role for log inserts; add archive job.

### B12. PHI access logs are incomplete
`PHIAccessLogger.php:15-20` only logs 8 named routes. Encounter views, prescription views, message threads, lab orders, documents, family members, telehealth sessions — **no log row written**. The route for `OperatorMemberController::search` was patched in the hardening commit but the same gap exists across the rest of the API. SOC 2 audit-controls failure. Fix: log every successful response on routes returning Patient/Encounter/Prescription/LabOrder/Document/Message.

## CRITICAL fixes — next 1–2 commits (high impact, low effort)

### C1. Auth perf N+1 (Performance + Backend)
`AuthController.php:487` — `userPayload()` runs `practices()->count()` per operator membership on every login + `/me` call. **Fix:** `with(['operator' => fn($q) => $q->withCount('practices')])`. **5-minute fix, every authenticated request faster.**

### C2. Idle session timeout never enforced
`Practice.session_timeout_minutes` exists in DB and is read by `AuditController::report()` (self-attestation only). **No middleware reads it; no token gets revoked.** HIPAA "automatic logoff" (164.312(a)(2)(iii)) gap. Fix: `EnforceIdleTimeout` middleware checks `last_used_at` on PersonalAccessToken vs setting; revokes on exceed.

### C3. Login throttle is IP-only
`/auth/login` is `throttle:5,1` per IP — distributed credential stuffing across IPs is uncapped. **Fix:** `RateLimiter::for('login', fn($r) => Limit::perMinute(5)->by($r->input('email').'|'.$r->ip()))` plus per-email lockout after 5 failures in 30 min.

### C4. Password reset flow does not exist
No `Password::sendResetLink`, no `forgot-password` route. Locked-out users have no recovery. SOC 2 8.2.1 control gap. Operationally pushes admins to share passwords.

### C5. MFA setup secret is client-trusted
`AuthController.php:332-352` — `setupMfa` returns secret; `enableMfa` reads `$request->secret` and trusts it. Phisher can substitute their own. **Fix:** `Cache::put('mfa-setup:'.$user->id, $secret, 600)`; `enableMfa` reads from cache, not request.

### C6. iCal feed unauthenticated, unthrottled, unlogged
`/api/calendar/ical/{token}` — leaked token = permanent PHI access (provider schedules with appointment reasons), no rate limit, no PHI access log entry. **Fix:** add `throttle:60,1`, log every fetch to `PhiAccessLog`, support rotation.

### C7. Operator addUser has no tenant scope
`OperatorController.php:128` — `email => exists:users,id` scans the entire users table. A malicious operator owner can attach any user from any practice on the platform to their operator scope, granting cross-tenant read. Fix: scope the `exists` rule to email + tenant in operator's owned tenants OR require an invitation flow.

### C8. Race condition: VisitPackCredit decrement
`UtilizationTrackingService.php:382` — `$credit->decrement(...)` in a loop without `lockForUpdate`. Two simultaneous claims pass the `>0` check and over-consume. Mirrors the Stripe webhook race we already fixed.

### C9. Backfill migration not chunked
`2026_04_29_000002_add_operator_id_to_practices.php` — backfill loops every practice in one transaction. With N=10K practices, 30K statements in one txn → Railway timeout. Fix: chunk by 500 per transaction.

### C10. Audit log writes from operator controllers stamp wrong tenant_id
`AuditLog` uses `BelongsToTenant`, which auto-sets `tenant_id` from `auth()->user()->tenant_id` — but the operator audit helpers in `OperatorController`/`MasterPlanTemplateController`/`TenantDomainController` may write to a different active tenant. Fix: explicitly pass `tenant_id` in audit row creation.

### C11. CSP header missing
`SecurityHeaders.php:11-26` sets X-Frame-Options + HSTS (production only) but **no `Content-Security-Policy`**, no `Cross-Origin-*-Policy`. XSS in any embedded widget can exfiltrate session tokens. Add `Content-Security-Policy: default-src 'none'; ...` for API responses.

### C12. CORS `supports_credentials: true` with token-only API
Tokens go in `Authorization` header, never cookies. `supports_credentials: true` tells browsers to send cookies that the API doesn't use — surface area without benefit. Fix: `false`.

### C13. Stripe webhook secret unconfigured = 500
`StripeWebhookController.php:56` — missing webhook secret returns 500 forever, Stripe retries indefinitely. Fix: fail-loud once, log, then disable the endpoint until reconfigured.

## IMPORTANT — fix this month

### Database
- **Add 14 missing indexes** (full SQL list in raw report, top: `(patient_id, status)` on `patient_memberships`, `(patient_id, encounter_date DESC)` on `encounters`, `(recipient_id, read_at) WHERE read_at IS NULL`, `pg_trgm` GIN indexes on `patients(first_name, last_name, email)` and `practices(name)`).
- **CareCoordinationController.php:239-260** — `overdue()` is 6N queries per active patient; rewrite as single GROUP BY.
- **NetworkMetricsService** — pulls full membership table for every analytics call; cache 5min in Redis or push to nightly snapshot table.
- **SoftDeletes missing** on `Message`, `PatientMembership`, `Invoice`, `Payment`, `LabOrder`, `LabResult`, `Referral`, `DispenseRecord`, `Document` — all HIPAA-relevant retention violations.
- **`providers.user_id` cascade on delete** — wipes a fired provider's entire chart history. Change to `nullOnDelete`.
- **`patient_memberships.plan_id` cascade on delete** — deleting a plan wipes membership + dunning history. Change to `restrictOnDelete`.
- **Encounter clinical fields not encrypted** (already in B2 above; called out separately because there's a parallel migration story).
- **`tenant_code` is varchar(6)** with 16M space and no max-attempt cap on the auto-generation loop. Bump to varchar(8), cap retries.

### Security
- **`DocumentController` MIME validation is by extension, no AV scan** — malware-staging vector. Switch to `mimetypes:` validation, force `Content-Disposition: attachment`, plan ClamAV/VirusTotal integration.
- **Cross-tenant validation gaps** in `BroadcastController` (custom audience patient_ids), `IncidentController` (provider_id), and several others — apply `Rule::exists('...', 'id')->where('tenant_id', $user->tenant_id)` everywhere.
- **`User.pin` stored unhashed** — kiosk PIN should be `bcrypt`.
- **CSS sanitizer regex blocklist incomplete** — misses `@charset`, `@font-face url(...)`, escape-encoded `expression`. Switch to AST parser (Sabberworm/CSS-Parser) or remove custom_css escape hatch.
- **Recovery codes only 32 bits of entropy** — `Str::random(4)` brute-forceable. Use `Str::random(10)`.

### Performance
- **Configure Redis on Railway** — sessions, cache, queue all on database driver. Single biggest latency drop (1 SELECT + 1 UPDATE on `sessions` per request today).
- **Lazy-load PracticePortal tabs** — 666 kB / 113 kB gzipped initial chunk. Convert 14 static tab imports to `React.lazy`. ~480 kB cut.
- **Move MOCK_* fixtures to lazy-loaded module** — `lib/api.ts` ships ~50 kB of fake patient data to every user in production.
- **Recharts loaded statically into PracticePortal** via EngagementSection/ProviderAnalyticsSection — wrap in `React.lazy` at PracticePortal import sites.
- **Queue the BroadcastController send loop** — currently blocks the request thread for 30+ seconds on a 500-patient broadcast.

### API contracts
- **Standardize error envelope**: 30+ controllers return `{error: ...}`, frontend `apiFetch` only reads `message` — silent error loss. Pick `{message, errors?, code?}`, refactor.
- **Standardize response envelope**: paginated endpoints return `{data: {data, meta}}` while non-paginated return `{data: [...]}`. Adopt API Resources, flatten to `{data, meta, links}`.
- **Money to integer cents** before opening API — currently `MembershipPlan.monthly_price` is decimal dollars but Stripe events flow as cents. Pick one before public launch.
- **Add `/v1` prefix** to all routes before any external consumer; emit `Deprecation:` and `Sunset:` headers per RFC 8594.
- **Add `X-Request-Id`** middleware for support traceability.
- **Fix HTTP method mismatches**: `PUT /messages/{id}/read` should be POST (state event, not idempotent replacement); `PUT /prescriptions/{id}/refill` reuses URL with method polymorphism — split into distinct sub-resources.

### Test coverage
- **Stripe Webhook handlers** — zero tests. Add coverage for signature verification, idempotency, replay, all subscribed event types.
- **Stripe payment flow** — `PaymentController`, `InvoiceController`, `ProrationService`, `DunningService`, `SmartRetryService` — zero tests. Revenue-critical.
- **Telehealth, Twilio SMS** — zero tests. Telehealth has a known bug with no regression coverage.
- **Move `StripeConnectServiceTest` from Unit → Feature** — uses RefreshDatabase + creates real Practice rows.
- **Frontend tests are trivial** (3 files, 12 tests). Add: AuthContext, api.ts 401 interceptor, OperatorContext header propagation, StripeConnectStatus component states, embedded widget event tracking.
- **CI gaps**: frontend Vitest never runs in CI; no coverage threshold gate; two redundant backend workflows.

### Backend code quality
- **`AuthController::register` is 160 lines, no transaction** wrapping multi-step writes (Practice → Operator → User → OperatorUser → Provider → mail → token). If Provider create fails, orphan User+Practice. Extract to `RegistrationService` in `DB::transaction`.
- **`SecurityHeaders` registered twice** in `bootstrap/app.php` (prepend + alias) — runs twice on aliased routes.
- **No real PHP enums** despite PHP 8.4. UserRole, MembershipStatus, AppointmentStatus repeated as magic strings 220+ times.
- **No FormRequest classes** for ~50 controllers. 121 inline `$request->validate(...)` calls.
- **Composer deps pinned to `*`** — `barryvdh/laravel-dompdf`, `laravel/sanctum`, `resend/resend-laravel`, `stripe/stripe-php`. `composer update` on a fresh checkout could ship breaking versions.
- **No global exception handler** in `bootstrap/app.php` — 403/404/422/500 use Laravel defaults, inconsistent shape.
- **5 services manually instantiated with `new`** in controllers — blocks DI/mocking. Use constructor injection.

### Frontend code quality
- **`PracticePortal.tsx`: 8,145 LOC, 121 useState, 45 `any`** — single file is a maintenance bomb. Split per tab (the `*Tab.tsx` pattern is already in use; the split stalled).
- **Color object duplicated 16 times** — `const C = {...}` in every portal/widget despite `colors` in design-system. Single `import { colors }` everywhere.
- **`toast()` helper duplicated 5 times** across files; design-system has a `<Toast>` component none of them use.
- **`Field`/`Input`/`Textarea`/`Toggle`/`ErrorPanel` duplicated 3+ times each** — promote to design-system.
- **React Query installed and provided but never called** — zero `useQuery`/`useMutation`. Either use it or rip it out.
- **`formatCurrency`/`formatNumber`/`formatMoney` duplicated 4 times with subtly different rules** — money displayed inconsistently across tabs. Move to `lib/format.ts`.
- **No AbortController anywhere** — components fire async then setState on unmounted. OperatorPortal tenant-switch racing is a real symptom.
- **3 `as never` casts in OperatorNetworkDashboard** to silence Recharts typing — replace with proper `Formatter<T, U>` import.

### Design / UX
- **22+ "Coming soon" buttons** (PracticePortal billing/staff actions, PatientPortal Edit/Change buttons, UserSettingsDropdown items) look interactive but do nothing. Make them disabled with tooltip or hide entirely.
- **OperatorPortal sidebar broken on mobile** — hard-coded `w-64`, no collapse/drawer. PracticePortal already has the right pattern; copy it.
- **Modals don't share a base component** — only `*Tab.tsx` modals use design-system `<Modal>`; PracticePortal inline modals (Create Plan, Edit Plan) lack Escape, click-outside, focus trap, body-scroll lock.
- **Form fields missing `htmlFor`/`id`** — only 23 across the 28 form-bearing files. Screen readers don't announce labels on focus.
- **Toggle switches lack `role="switch"` + `aria-checked`** — invisible to assistive tech.
- **Patient-facing copy leaks clinical jargon** — PHQ-9/GAD-7 acronyms, ICD-10 codes shown to patients in mono badges, prescription strings like "Sertraline 100mg, Bupropion 150mg XL". Rewrite for patients.
- **Dark mode is essentially unimplemented** — `useTheme` exists but only `HeaderToolbar` actually branches on `isDark`. Toggling theme has zero visible effect on portal bodies.
- **No empty/skeleton states**: every list uses `<Loader2 spin>`. Design-system has `<Skeleton>` — used by exactly 0 portals.

### HIPAA / SOC 2
- **No accounting-of-disclosures endpoint** (164.528) — patients can't see who viewed their record.
- **No patient data export endpoint** (164.524 right of access) — required within 30 days of request.
- **MFA not enforced** — `Practice.enforce_mfa` flag exists but no middleware blocks login when role=practice_admin and `mfa_enabled=false`.
- **No security incident model** (clinical Incident exists; no security counterpart). No IR runbook.
- **No anomaly detection** on `PhiAccessLog` (e.g., user reading >50 charts/hour).
- **DOB returned in cross-tenant operator search results** — Safe Harbor identifier. Drop or year-only.
- **BAA status unknown** for Resend (transactional email going to patients), Sentry (PHI may end up in stack traces), Twilio (HIPAA tier?), Daily.co (Scale plan only). Audit each, scrub Sentry payload keys.
